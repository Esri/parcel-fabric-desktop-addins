Fabric points

Points not connected to lines

Every line in the parcel fabric must have two points at each end that it references using the FromPointID field and the ToPointID field. When points exist in the fabric that do not have associated lines, they are considered orphan points. Orphan points may have occurred as a result of problems in the earlier software releases that have since been addressed.

This fabric scan process will first build a list of all the points and their object ids from the fabric points table. The scan will then check the fabric lines table for each line records FromPointId and ToPointID value, and will remove the ids that it finds from the first points list. Any points that remain in the original points list are reported as orphan points and may, optionally, be deleted. The report presents an IN clause SQL statement that may be copy and pasted for use in the fabric points sublayers definition query, or for use in the select by attributes command for the points.
