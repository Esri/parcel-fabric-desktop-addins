Fabric Line points

Focused fabric scans for inconsistencies in line points

There are 2 different types of focused scans that can be used for line points:

1.       Line points with invalid references

2.       Line points with displaced geometry

Line points with invalid references

This scan searches for inconsistencies in each line points attributes and tests for the following problems:

1.       FromPointID is the same as the LinePointID

2.       ToPointID is the same as the LinePointID

3.       FromPointID is the same as the ToPointID

4.       Orphaned line points, where the combination of FromPointID and ToPointID on the line point does not have a matching line in the fabric lines table with these same point ids.

Any problem line points with these conditions are reported and can be deleted, if you choose to.

Line points with displaced geometry

This scan searches for geometry inconsistencies between each line points geometry and the geometry of the stored fabric points. A line point should always be at the same location as the fabric point. The scan tests for

1.       line points that have a point geometry that does not match the fabric point location referenced.

Any line points with these conditions are reported and can be deleted if you choose.

This second line point scan can also be configured to write the offset distance to all line point records in the fabric line point table. The offset is computed perpendicular to the line between the referenced 'From' and 'To' points stored in the fabric points table.

 

In most cases the line point should be close to the straight line computed between these points. However there are varying degrees of acceptability for how large this quantity can be before it is considered a bad line line-point.

To write line offsets for all line points add a field called 'LINEOFFSET' to the line points fabric table. The field must be of the type <Double>.

 

 

Once these values are present on the line points table, you will be able to make more informed decisions about any line points that may need to be selected and removed using the Delete Selected Line points tool on the Delete Fabric Records toolbar, installed as part of this add-in.

 

The following sections provide some more background on line points.

How are line points used?

Line points are used for line cartography; specifically, they are used to prevent gaps and slivers between parcels in places where parcel corners fall near to lines from other parcels.

In this image the red line in the magnified view is the southern boundary of Lot 21 and it extends between the points A and B.

It shows an example of three lines overlapping the red line and creating a gap and an overlap. In this image no line points have been added to the data. When line points are added they are placed on top of the existing fabric points; a line-point does not replace the pre-existing point.

You can choose to have line points fix gaps and slivers in one of 2 different ways, using either standard line points, or flex line points.

Standard line points

1.       Standard line points are moved to be placed on the straight line between the terminal points. From the example above, standard line points for Lot 21 keep the line AB that belongs to Lot 21 straight, and move the point J (shared by Lots 5 and 6) and point K (shared by Lots 4 and 4) onto line AB:

 

Flex line points

2.       Otherwise if the line points created on top of J and K are set as Flex points, then they are not moved, but the straight line between the terminal points is flexed to fit to the line point positions; the straight line is bent to fill the gaps, and remove the slivers. From the same example above, the flex line points for Lot 21 would bend the line AB, by flexing the line to fit J and K:

Whether a line point is a flex point or a standard point is determined by the choices that you make while editing. One example is when using the Add Line Point tool to create a line point:

While using this tool, if you have the option to bend checked on, then a flex line point is created. Otherwise, if you UN-check the option to bend the line, then a standard line point is created.
