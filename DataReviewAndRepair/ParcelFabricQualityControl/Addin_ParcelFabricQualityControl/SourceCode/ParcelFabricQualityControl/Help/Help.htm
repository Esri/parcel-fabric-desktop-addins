Parcel Fabric Quality Control Add-in

The parcel fabric quality control add-in helps you visualize quality indicators on parcels. It also provides tools to re-calculate distance and direction attributes based on the geometry of the lines, and based on the filters that you define. Directions and distances can be calculated independently of one another, and you can filter the lines that will be updated based on your allowable difference tolerance between the geometry and the current attribute.

You can choose to report the results in a list, review the updated symbology, and elect to keep the edits, or undo the results without making the changes to the database. If you provide an elevation source layer in the map, the combined scale factor will be applied for distance computations.

Coordinates versus Record Measurements

Coordinates in the fabric are changed through computation via the fabric adjustment that uses the directions (also called bearings) and distances stored on the lines; the stored directions and distances of the lines are not altered when moving parcel points in this way, only coordinates are changed. Coordinates of fabric points may also be changed using tools such as the transform toolbar. Just as with the fabric point adjustment, the attribute bearings and distances are not altered when using the transform toolbar on fabric points.

Add Quality Control Layers

Click the Add Quality Control Layers button on the toolbar to add a group layer to the map that contains layers symbolized to show:

        Parcel misclosure

        Parcel distortion

        Significant difference between line shape length and attribute distance

 

Parcel Misclosure

 

 

Misclosure on the parcels is a metric of how well the directions and distances for each parcels lines close mathematically. It is independent of the parcels current coordinates, and remains unchanged by coordinate changes. Misclosure is reported in the fields on the parcel table called MiscloseDistance, MiscloseBearing, and MiscloseRatio. It can only be changed by making changes to the directions and distances on the parcel lines. Ideally this would be achieved by making sure the values stored match the original values from the recorded documents. This information is not always easy to update if it had not been captured and stored in a digital form at the time of initial entry.

Parcel Distortion

Coordinates and measurements do not always correlate with one another but the relative distortion between coordinates and measurements can be minimized. This distortion is reported in fields on the parcel table called ShapeStdErrorN and ShapeStdErrorE. Distortion is calculated by comparing a parcels measurements to its current coordinates, and these values change based on changes to coordinates.

Minimizing the distortion values in these fields is the goal, and can done by updating fabric point coordinates through the fabric adjustment, but it requires good record information, and control points. The parcel distortion values do not have meaning if the parcel misclosure is high. If a parcel has a high misclosure (see previous section), then that should be considered and addressed prior to making an assessment of parcel distortion.

Distance Difference (shape length minus attributed distance)

This lines layer shows and labels any lines that have a difference between shape length and attribute distance of more than 0.3 feet or 0.1 meters.

Since the fabric distances are represented as ground values, whereas the shape length is defined in a projected coordinate system (grid) at approximate sea level, this pre-defined difference can be exceeded for longer lines so it is to be expected that longer lines may show up in this layer, but may not necessarily need further analysis depending on the difference displayed.

Note that after clicking the button and adding the default Quality Control Layers, you can re-configure the definition queries, label expression and other layer properties to your liking and save the map document. You can also define your own layers: start by expanding the fabric dataset node in the Catalog window and dragging the fabric class into your map document.

Overview of Inversing: Calculating direction and distance from geometry

When the stored shape or geometry of single segment lines are used to populate the bearing and distance attribute fields, the direction and distances are inversed from the coordinate locations. The process of computing coordinates from record values is turned upside down and instead the record values are calculated from coordinates, hence the term inverse.

Record distances and directions -> Compute Coordinates (Direct)

Stored Coordinates -> Compute Distances and directions (Inverse)

Oftentimes the bearings and distances attributed on the parcel lines have been previously calculated from the geometry, such as during the data loading process in which the original source data did not have any COGO attributes. In these cases the parcels will start out with zero misclose and no distortion, and the distances will be represented as grid distances and not ground distances.

In other cases the original COGO attributes were present on the source data when imported into the fabric, but were not current with the geometry that they were attached to. In these cases many of the parcels have a poor misclose but it is caused only by one or two lines in each parcel, while all the other lines in the parcel have correct record values.

To handle the variety of cases the inverse tools provide different options, as described in the next sections.

Inversing Distances

The Distance Inverse tool works on a selection and requires you to be in an edit session. Typical use is based on a selection of parcels; choosing to select directly from the QA Layers described above helps to isolate the parcels that need to have distances inversed. The tool will also work on selected lines.

After making a selection, click the Distance Inverse button on the toolbar. The Distance Inverse dialog appears:

 

The dialog contains three main parts, first the choice of how to scale the distances computed from the geometry, second an option to exclude lines from being inversed based on how different the attribute is from geometry, and third a choice to report the results.

The default settings are shown above though it is recommended that an approximate height above sea level is used. This elevation only needs to be an average or approximated for the area of the selected parcels.

Inversing Directions

Like the Distance Inverse, the Direction Inverse also works on a selection and requires you to be in an edit session. Typical use is based on a selection of parcels; choosing to select directly from the QA Layers described above helps to isolate the parcels that need to have directions inversed. The tool will also work on selected lines.

After making a selection, click the Direction Inverse button on the toolbar. The Direction Inverse dialog appears. The image below shows the default settings:

 

 

The Direction Inverse dialog has three sections, one for defining the direction offset, the second for excluding lines that are within the difference tolerances that you specify, and the third a choice on whether to report the results.

 

The first section allows you to directly enter the direction offset, or to access it from the Editor options, by clicking the right-facing arrow button.

 

 

In the first section the recommended (default) approach is to use the option that applies the direction offset based on each parcels lines. This approach will analyze the lines, and look for a consistent offset, and recomputed the lines that have an offset that is not consistent with the others. This means that the Direction inverse will preserve the parcel rotation for each parcel. If the original record directions are based on parcels with different basis of bearings, then this option will continue to preserve the original record and honor the different parcels varying rotations/basis of bearings.

 

The second section for defining exclusions includes 2 options.

 

 

The first is for comparing the geometry computed direction with the attribute direction. The difference between them is an angle, and if that angle is less than the specified amount, in seconds, then it is not inversed.

The second filter is for defining the lateral distance offset of the angular difference (distance subtended.) This varies based on the length of the line, meaning that for long lines a small angular difference will translate to a larger lateral distance offset difference than it would for shorter lines.

 

The lines are excluded if either one of these tests is true.

Once you have made the choices for the direction inverse, click the Calculate button. A report is displayed showing how many of the selected parcels were affected, and how many of their lines were inversed.

 

 

The report also shows, in decreasing order of lateral offset, the list of lines that had changes applied, including the change in bearing, expressed as an angle, and the lateral effect (distance subtended.)

 

Coordinate Inverse

The Parcel Fabric Quality Control Add-in also provides a method to inverse the attribute values for parcel fabric points from the stored geometry. The command is accessed by right-clicking the fabric dataset in the catalog window and clicking Coordinate Inverse. Coordinates should always match the geometry, the command will first check to see if there are any differences and update any points that have a coordinate inconsistency.
