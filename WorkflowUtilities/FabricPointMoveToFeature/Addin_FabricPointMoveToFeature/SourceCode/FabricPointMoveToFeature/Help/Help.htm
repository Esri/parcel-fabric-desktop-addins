Fabric Point Move to Feature Add-in

 

The Fabric Point Move to Feature add-in provides methods to update the positions of parcel points based on feature geometry locations. Feature layers are used as a target reference, and contain the features that are used to update the fabric points.

Fabric points can be updated using either a line layer or a point layer.

 

Reference feature layer

The reference feature layer is used in combination with the Move points button. You choose the reference layer to use and then press the Move points button to apply the fabric point updates based on the references. More information about creating and using reference layers appears below.

Move points

The Move points button updates the location of the referenced fabric points. This updates the positions of the parcels, connected lines, and the line-points. The edit is applied in an edit operation. If the result is not desirable, click Undo to revert the data to its previous state.

 

Configure

Press Configure to access the reference settings. The Configuration dialog four tabs that allow you to define the method you want to use to move points, selection settings, reporting results, and conditions for merging fabric points.

Creating and using a reference point layer: point to point matching

The toolbar will detect a layer as a candidate point reference for fabrics if it has the following characteristics:

        Simple point feature class in a geodatabase

        Field of type <Long> or Field of type <Guid>

 

Layers with these characteristics are listed in the toolbars drop-down for use as a reference.

 

 

In the Method tab of the Configuration dialog you check the option Use point to point matching and then in the drop-down list choose the field that the Move points tool should use to look for matches. For simplicity its recommended that reference layers are added to the map one at a time.

 

Fabric points are moved to the reference point features in the reference layer if the reference point has in its <Long> field the Object ID of the fabric point. This edit happens when pressing the Move points button.

Reference points may be created by any means available, for example:

        Manually create new points in the reference layer, and enter the fabric points Object IDs, or their Guid values to the reference points.

        Run geo-processing tools to make a copy of fabric points into the reference layer, then use spatial adjustment methods to update the reference point positions, keeping the original fabric points ids on the reference points.

 

 

Creating and using a reference line layer: line-start to line-end matching

The toolbar will detect a layer as a candidate line reference for fabrics if it has the following characteristics:

        Simple line feature class in a geodatabase

        The add-in configuration is set to work with lines.

 

 

After updating the configuration, the layers with these characteristics are listed in the toolbars drop-down for use as a reference.

 

Fabric points are moved to the end point locations of the reference line features if the start point locations of the reference lines exactly match the fabric point location. This method differs from the point to point approach in that it is not dependent on attribute information in the reference feature.

 

This edit happens when you press the Move points button.

 

 

Reference lines may be created by any means available, for example:

        Manually create new lines in the reference layer.

        Run geo processing tools.

 

Preventing small changes in point positions

In some cases you may have a large number of reference features that represent only small changes in location of fabric points. This may result in a large number of parcel edits that are not significant from a practical point of view, but that have none-the-less affected a large number of records in the database. This can be prevented using the tolerance on the Method tab.

Check Do not move fabric points for changes less than: and then enter a distance value to represent how much a reference change can be before it comes into effect and moves a fabric point.

For example if you enter a value of 0.01m, then any reference feature point that is closer to its fabric point than 0.01m, it will not be moved.

Choosing a datum transformation

When the parcel fabric has a different geographic datum to the reference feature class, then a datum transformation is used to project the reference data into the same coordinate system as the fabric. A transformation is automatically chosen for you unless you configure the tool to present a list of available datum transformations. To configure this behavior check the option Prompt for a datum transformation when the datums differ.

Selected Feature Options

 

In the configuration dialog, the Select Feature Options tab allows you to choose which referenced fabric points should move based on your selection.

Ignore Selections

You can choose to move any fabric points that have references, regardless of the current selection. For this behavior, check the option Ignore selections and move all fabric points with references.

Use Selected Reference Features

If you want to only move fabric points of the reference features that are selected, then check the option that says Move fabric points of the selected reference features.

The image shows three reference points, and two of them are selected. Since the tools configuration is set to work on selected reference features, only the two selected point references have their corresponding fabric points moved, while the unselected reference point does not affect its fabric point.

Use Selected Parcels

If you want to configure the tools behavior to move the fabric points based on selected parcels, then check the option Move fabric points of selected parcels.

The selected parcel in this image has three fabric points with point references. Since these three points are attached to the selected parcel, they are all moved to their referenced locations.

Configure behavior for when there is No Selection

If you have chosen to base fabric point changes on selected features, then there is also a secondary choice if you run the Move Points tool without a selection. You can configure the tool to either give a message that no references were found, or to offer the option of processing the reference features in the map extent.

If you would like to be prompted to process data in the map extent, then check Prompt for choices when there is no selection

Reporting

In the configuration dialog on the Reporting tab you can choose to present a dialog of the results of fabric point changes. Check Report fabric point changes if you would like to see these results after running the tool.

The results are presented showing the ids of the points, and the bearing and distance of the change, ordered by the longest distance change to the smallest distance change.

 

Merging Point Options

In the configuration dialog on the Merge Point Options tab, you can configure behavior for cases of fabric points being moved into close proximity of one another.

To activate this behavior check Merge referenced fabric points that become grouped within: and set the grouping tolerance.

In this example the tool is configured to merge points that are within 1.0 meter. After the tool has run, if there are any fabric points that do not have a parcel line connected between them, and are less than 1.0 meter apart, then the fabric points will be merged, meaning that one of the points will be kept, and the other will be removed. All the lines that originally connected to the removed point will now instead connect to the point that was kept.
